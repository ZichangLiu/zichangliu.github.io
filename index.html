Dynamic Programming

Posted on 2020-11-08 In Algorithm Disqus:

Methodology

Core Idea

DP的核心思想类似于数学归纳法:

  1. 把一个大问题(size == n)的解决方案用比他小的问题们来解决, 也就是思考从问题size = n-1增加到size = n 的时候, 如何用小问题的solution构建大问题的solution
  2. 就是利用历史记录，来避免我们的重复计算。而这些历史记录，我们得需要一些变量来保存，一般是用一维数组或者二维数组来保存。
  3. 与recursion的关系:
    • Recursion从大到小来解决问题, 不记录任何sub-solution, 只要考虑
      • base case
      • recursive rule
    • DP从小到大来解决问题, 记录sub-solution
      • 由size(< n)的subsolution (s) -> size(n) solution
      • base case
      • induction rule

Introduction

In Java, String is class.

Popular implementation: char[] array

  1. ASCII representation of a letter (0 ~ 127 (2^7-1))
  2. Unicode: the latest version of Unicode contains a repertoire of more than 110,000 characters covering.
    • 对于ASCII里127号以下字符, Unicode维持原编码, 只是将长度由原来8位扩展为16位, 而其他语言的字符则采用全部重新统一编码.
    • Java推出时即采用了Unicode, 所以Java的char类型是2个byte, 即16bit
    • 由于英文符号只需要用到低8位, 所以其高8位永远是0, 造成了空间上的浪费
  3. Character set VS. Character encoding
    “In there two terms, ‘set’ refers to the set of characters and their numbers (code points), and ‘encoding’ refers to the representation of these code points”
    Encodings: UTF-8, UTF-16

    There is no need for maintaining serveral copies of String objects with the same literal, since String objects are immutable.
