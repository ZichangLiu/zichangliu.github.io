1. Basic Concepts

N1 -> N2 -> N3 -> N4 -> … -> N1000 -> NULL

链表问题都是inplace的, 所有结点在内存中都是原地不动的, 变化的只有结点里存的下一结点的地址

Key points:

  1. When you want to dereference a ListNode, make sure it is not a NULL pointer
    //ListNode p = new ListNode(10);
    p.value
    p.next.value //“.”is an action that is called de-reference
  2. Never ever lose the control of the head pointer of the linkedList, or you cannot get access to other nodes anymore.

DummyHead:
When we want to build a LinkedList from scratch (initially zero nodes), in order to avoid null pointer dereference (NPE), we need to use a dummy head.
ListNode dummy = new ListNode(0);
dummy.next = N1;
return dummy.next;
Tail:
When we want to append a new Node to the end of the LinkedList, we need to maintain a tail pointer, besides the header at the beginning of the algorithm.
tail.next = cur;
tail = cur;
//DummyHead is a new node, but tail is not a new node. Tail can be treated as a pointer that stay at dummyHead or current node.

2. When to use Iterative way / Recursive way

a. In real work environment, consider using iterative way first to avoid call stack overflow.
b. In interview environment:
i. for tree related problems, usually use recursion way is more preferred.
ii. for other problems. you can evaluate the real problem:
1. if you can use both methods are their time complexity are the same, then you can use either way.
2. if their time complexity are different (Fibo: recur vs dp), pick smaller one.

3. Classic questions:

  1. Reverse a linked list(a.Iterative way / b.Recursive way)

  3. Check if linkedlist has a cycle
  4. Insert in sorted linkedlist
  5. Merge two sorted linkedlist O(nlogn)
