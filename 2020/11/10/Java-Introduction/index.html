Java Introduction

Posted on 2020-11-10 In Java Disqus:

Introduction

Wikipedia:

Java is a set of computer software and specifications developed by James Gosling at Sun Microsystems, which was later acquired by the Oracle Corporation, that provides a system for developing application software and deploying it in a cross-platform computing environment.

Features of Java

C++:
The source code needs to be compiled to directly the machine instructions on which machine the program is running on. The compiled byte code is different on different machines. (machine need to recognize the compiled code)
Python:
Interpreter: Python
Compiler: pyc
Java:
The source code just need to compiled once and it can run on any machine with JRE installed. (JRE, not the machine need to recognize the compiled code)

  1. Java: Platform compatible, write once, compile once, run everywhere on JVM
    C++: write once, compile many times but everywhere
  2. JVM: Java Virtual Machine
  3. JRE vs. JDK
    • JRE (Java Runtime Environment) is the JVM program, Java application need to run on JRE.
    • JDK (Java Development Kit) contains the tools for developing Java programs running on JRE. Especially, it provides the compiler “javac”.
    • JDK = JRE + Devemopment Tools
    • JRE = JVM + Libraries Classes
    • Compiler – 编译器
    • Interpreter – 解释器
  4. Platform compatibility: language virtual machine <- Java, Python, JavaScript….
    Better performance: compile to native language (machine language) <- C, C++, Go
  5. Compile/Interpret language:
    • Java source code -> JVM readable byte code (.java -> .class)
    • JVM interprets the Java bytecode to pre-compiled routine on machine code.
      or
    • Just-in-time (JIT) compilaion to compile the bytecode to native instructions. (类似于cache)

      The current version of Sun HotSpot JVM uses a technique called Just-in-time (JIT) compilation to compile the bytecode to the native instructions understood by the CPU on the fly at run time.
      Some implementations of JVM might interpret the bytecode instead of JIT compiling it to machine code and running it directly. While this is still considered an “interpreter.” It’s significantly different from interpreters that read and execute the high level source code.

  6. Interpreter to routines on systems with different machine codes.
    Compile/Interpret Language?
  7. Strongly encouraged Objected
  8. All types (reference types, primitive types) are always pass by value
  9. Java does not support unsigned integers
  10. Pointers vs. References.
    • Java has no pointers arithmetics
    • Difference between pointers in C++ and references in Java:
      • references in Java are strongly typed
        String s = “abc” // here s is a reference of type “String”
      • there is no pointer arithmetic on references
        Pointers in C++ are much more powerful but also more dangerous.
        E.g.
  • 强类型: 每个变量和对象都必须声明类型. 在编译时确定好类型了. 执行时不能更改。偏向于不容忍隐式类型转换。
  • 弱类型: 编译时不确定类型，执行时根据语义确定是哪种类型。通俗地说，弱类型偏向于容忍隐式转换，把不同类型的变量放在一起运算。
  • 静态类型: 编译的时候就知道每一个变量的类型，因为类型错误而不能做的事情是语法错误。
  • 动态类型: 编译的时候不知道每一个变量的类型，因为类型错误而不能做的事情是运行时错误. 比如说你不能对一个数字a写a[10]当数组用。
  1. No operator overloading “+”, “-“…
  2. Classes/Objects are always allocated on the Heap, there is no way to allocate objects on Stack. (“new”)
  3. Garbage Collection
  4. Single inheritance, multiple inheritance only be done by implementing multiple interfaces. Java enforce single inheritance to eliminate this kind of problems and to make the class hierarchy much cleaner and easier to maintain.
    Allowing multiple implementation of different interfaces to provide similar support of “multiple inheritance”.

Advantages of Single Inheritance:

  1. 所有Object都有Object class属性 - global root
  2. handles Ambiguity

More Discussion:

  2. 比如网站, 瓶颈在于网络传输

  4. 所以用的人多, 所以开源环境好, IDE好用框架优
