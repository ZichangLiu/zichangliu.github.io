Exceptions

Posted on 2020-11-10 Edited on 2020-11-11 In Java Disqus:

Overview

An Exception indicates conditions that a reasonable application might want to catch. An Error indicates serious problems that a reasonable application should not try to catch.
Throwable Class graph:

  • Error
  • Exception
    • IOException (EOFException, FileNotFoundException, MalformedURLException, UnknownHostException)
    • ClassNotFoundException
    • CloneNotSupportedException
    • RuntimeException
      • ArithmeticException
      • ClassCastException
      • IllegalArgumentException
      • IllegalStateException
      • IndexOutOfBoundsException
      • NoSuchElementException
      • NullPointerException

More Details

RuntimeException/Unchecked Exception

  1. RuntimeException: is a subclass of Exception. No need to catch it if you do not want to. ArithmeticException, ArrayStoreException, ArrayIndexOutOfBoundsException, ClassCastException, etc
  2. Checked Exception: Exceptions other than RuntimeEcception. Must use either try/catch block or throws.

    Java认为CheckedException都是可以被处理的Exception, 所以Java程序必须要明确地处理Checked Exception. 如果程序没有处理CheckedException, 那么程序在编译时就会发生错误无法编译. 这体现了Java的设计哲学: 没有完善错误处理的代码根本没有机会被执行
    Guideline: if a client can reasonably be expected to recover from an exception, make it a checked exception. If a client cannot do anything to recover from the exception, make it an unchecked exception.

Finally

Finally will always be executed (without exception or not), even with a return. (But not System.exit(0))

Throw vs. Throws

  1. Throw:
    The throw keyword in Java is used to explicitly throw an exception from a method or any block of code. General form of throw is: throw Instance;
    But this exception i.e, Instance must be type Throwable or a subclass of Throwable
    Primitive data types such as int, char, floats and non-throwable classes cannot be used as exceptions
  2. The flow of execution of the program stops immediately after the throw statement is executed and
    • the nearest enclosing try block is checked to see if it has a catch statement that matches the type of exception
      • If it finds match, controlled is transferred to that statement
      • otherwise next enclosing try block is checked and so on.
        • if no matching catch is found then the default exception handler will halt the program
  3. Throws:
    • Throws is a keyword in Java which is used in the head of method to indicate that this method might throw one of the listed type exceptions.
    • The caller to these methods has to handle the exception.
      e.g. type methodName (parameters) throws exceptionList {}
      Here, the exceptionList is a comma separated list of all the exceptions which a method might throw.
    Throw is an action, throws is an indicator of a function.

try-catch-finally规则

  1. 必须在try之后添加catch或finally block. try后可同时接catch和finally block, 但至少有一个block
  2. 若代码同时使用catch和finally block, 则必须将catch块放在try块之后
  3. catch block于相应的Excetion Class的class type相关
  4. 一个try block可能有多个catch block. 这种情况下, 总是执行第一个匹配的block
  5. 在try-catch-finally结构中, 可重新抛出Exception
  6. try-catch-finally结构可以嵌套
  7. 除了以下情况, 总是执行finally做为结束:
    • JVM过早终止(调用System.exit(int))
    • 在finally抛除一个未处理的异常
    • 计算机断电、失火或遭遇病毒攻击

四种处理Exception方法:

  1. Re-throw:
    If you run it, Java program will stop, and top level exception will be thrown
  2. Multiple catches
  3. Multiple catches in Java7
    catch (IllegalAccessException | NullPointerException) {}
    • Note: This is “|”, not “|”. This is just a syntax rule.
  4. Catch a parent class:
    This is okay. Because you are using “base” class. Both NullPointerException and IllegalAccessException are subclasses of Exception.

Do not catch Throwables (avoid catching errors)
