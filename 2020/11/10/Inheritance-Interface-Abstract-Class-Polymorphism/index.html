Inheritance, Interface, Abstract Class, Polymorphism

Posted on 2020-11-10 In Java Disqus:

Inheritance(继承):

Base class(父类), Derived class(子类)
Definitions:
A class that is derived from another class is called a subclass (also a derived class, extended class, or child class). The class from which the subclass is derived is called a superclass (also a base class or parent class).

Override vs. Overload

Definitions:

Override is when you redefine a method that has been defined in a parent class (using the same signature). Resolved at runtime.
Overload is when youy define two methods with the same name, in the same class, distinguished by their (different), Resolved at compile time.

  1. 哪些函数能够调用取决于reference (声明)的类型 – compile time
    对于被Override的函数, 调用哪个版本, 取决于object(实现)的类型 – runtime
  2. Reference类型决定什么可以调用, Instance类型决定调用的具体方法的行为
    • 先看Reference决定使用哪个函数的signature
    • 再看implementation决定使用哪个函数版本(function body)
  3. Some thoughs:
    1. ALWAYS use @Override when overriding.
    2. Use result of example programs to make judgement, not by your intuitions.
    3. Naming is important.
  4. JVM调用方法的顺序
    • 选定method
      先看reference类里是否有对应方法 – compile time
      • 如果有精准匹配的(精准匹配的可能直接来自父类如果没有override), 暂时决定调用这个方法
      • 如果没有, 看看用cast是否能将找到对应方法, (cast到最近可用的类型: 尽量cast成父亲而不是爷爷), 如果有, 那么compile通过, 暂时决定调用这个方法
      • 如果还没有, compile error
    • 选定版本(一旦到了runtime, method不能再改, 只能选版本)
      • 在runtime中, 真正调用的是instance实现类的对应方法(包括所有从其父类继承而来的方法), 如果instance类中override了之前在compile时暂时决定调用的方法, 则真正调用的是这个override后的方法.

Interface and Abstract Class

Definitions:

A class must be declared abstract when it has one or more abstract methods.
A method is declared abstract when it has a method heading, but no body, which means that an abstract method has no implementation code inside like normal methods do.

Difference:

  1. Java does not allow multiple inheritance. In Java, a class can only be derived from one class, whether it’s abstract or not. However, a class can implement multiple interfaces. Java does not allow multiple extends, but a class can implements multiple interfaces.
  2. An abstract class has one or more abstract methods, can have some non-abstract methods, constructors and instance variables as well, can have non-public members.
    An interface has only abstract methods. All methods must be public. Any class that implements the interface is responsible for providing the method definition/implementation.
  3. abstract class: is-a …
    interface: has-an-ability of …

Related Interview Question:

When will you use abstract class vs. interface?

  1. An abstract class is good if you think you will plan on using inheritance since it provides a common base class implementation to derived classes.
  2. An abstract class is also good if you want to be able to declare non-public members. In an interface, all methods must be public.
  3. If you think you will need to add methods in the future, then an abstract class is a better choice. Because if you add new method headings to an interface, then all of the classes that already implement that interface will have to be changed to implement the new methods.
  4. Interfaces are a good choice when you think that the API will not change for a while. Interfaces are also good when you want to have something similar to multiple inheritance, since you can implement multiple interfaces.

不能直接instantiate abstract class or interface

e.g. Employee e = new Employee(); // WRONG
In Java, for your base class, Interface vs. Abstract Class vs. Concrete Class

  1. An abstract class provides a common base class implementation to derived classes. Use abstract class (or even concrete class) for base class if the derived classes share commmon implementation.
    e.g. base class: House
    derived class: TownHouse, SingleFamilyHouse
  2. Interface helpes you (or enforces you) to focus on the API signature definition
    e.g. interface Collection: add(E), size(), contains(Object)…
    Collection’s subclasses may share NO common implementation at all.
  3. Multiple inheritance: implement multiple interfaces
    e.g. ArrayList implements:
    Serializable, Cloneable, Iterable, Collection, List, RandomAccess
  4. The relationship between the interface itself and the class implementing the interface is not necessarily strong (capability)
    e.g. class House implements AirConditioning
    ArrayList implements Serializable

Polymorphism(多态) and Overriding

Override:

a subclass or child class to provide a specific implementation of a method that is already probided by one of its superclasses or parent classes.
Override <- run-time vs. Overload <- compile time

Polymorphism:

A call to a member function will cause a different function to be executed depending on the type of object that invokes the function. (subtype polymorphism)
