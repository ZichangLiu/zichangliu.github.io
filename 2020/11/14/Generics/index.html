Generics

Posted on 2020-11-14 Edited on 2020-11-15 In Java Disqus:

Defination: Types are parameters (Types can be various)

Java Generic methods and generic classes enable programmers to specify:

  • with a single method declaration, a set of related methods, or
  • with a single class declaration, a set of related types

A type or method to operate on objects of various types while providing compile-time type safety
(vs. runtime safety: too late, exceptions)
Compiler would make sure the type is right during the compile-time (not runtime) so as to remind you before running this application.

You can write a single generic method declaration that can be called with arguments of different types. Based on the types of the arguments passed to the generic method, the compiler handles each method call appropriately.

  1. Syntax: All generic method declarations have a type parameter section delimited by angle brackets (<>) that precedes the method’s return type:
    public static E method(E[] input)
  2. Number of Generics: Each type parameter section contains one or more type parameters separated by commas:
    public static <A, B, C> void method(A[] array, List list, Queue queue)
  3. Applicable Uses:
    • member variable
    • method parameter
    • method return type
    • method local variable
  4. Object only: A generic method’s body is declared like that of any other method. Note that type parameters can represent only reference types, not primitive types (like int, double and char)

Advanced Feature

  1. From a syntax point of view: E can be any object, < may not compile
  2. From a logical point of view: To find the minimum, E should be comparable. If it is a random Object type, < may not make sense.
  1. There may be times when you’ll want to restrict the kinds of types that are allowed to be passed to a type parameter.
  2. For example, a method that operates on numbers might only want to accept instances of NUMBER or its subclasses. This is what bounded type parameters are for.
  3. To declare a bounded type parameter, list the type parameter’s name, followed by the extends keyword, followed by its upper bound.

Generic Class: Only the class needs to be parameterized: its method function does not need anymore
When you create object:
a. Node<Integer> node = new Node<>(1);
b. Node<Integer> node = new Node(1); //is not good (Warning)

Java泛型实现原理: 类型擦除保留原始类型(Type Erasure)
从类型擦除可以看出:

  1. 必须是Object继承的类型作为泛型类型, 不然没法转成Object
  2. 运行期类型检查职能看到原始类型, 如ArrayList只能看到ArrayList不能看到Integer
  3. 新建一个泛型类指定的对象是不可以的, 比如: K temp = new K();
  4. 编译器自动加入类型的强制转换, 比如:
    ArrayList<Employee> emps = ...;
    Employee e = emps.get(0);
    -> 先获取Object对象, 再类型转换成Employee

Subclass and Superclass in Generics
A subclass can always be casted to a superclass but not the other way
In Generics, it’s different:

If you allow fruits = apples, then we may do fruits.add(new Orange()), but Orange is not Apple
Generics check the type during compiling time. In contrast, Array does not
So when we want to use a super/subclass when working with generic references, we do:
<? Extends T> or <? Super T>

通配符的使用(Wildcard)
? extends (Upper Bounded Wildcard)
? super (Lower Bounded Wildcard)
Why? It only knows apples is a list of objects that are superclass of Apple, but not which superclass in the inheritance tree.
Conclusion: To get data use <? extends>, to write data use <? super>

The general rule is to use wildcards when you can because code with wildcards is generally more readable than code with multiple type parameters. When deciding if you need a type variable, ask yourself if that type variable is used to related two or more parameters, ot to relate a parameter type with the return type. If the answer is no, then a wildcard should suffice.

泛型类型与通配符使用比较:
泛型类型: 有个类型叫T, 可以用T干很多事情
通配符: 我不知道的一个类型, 不能直接用?, 需要类型捕获之后用
