Introduction

In Java, String is class.

Popular implementation: char[] array

  1. ASCII representation of a letter (0 ~ 127 (2^7-1))
  2. Unicode: the latest version of Unicode contains a repertoire of more than 110,000 characters covering.
    • 对于ASCII里127号以下字符, Unicode维持原编码, 只是将长度由原来8位扩展为16位, 而其他语言的字符则采用全部重新统一编码.
    • Java推出时即采用了Unicode, 所以Java的char类型是2个byte, 即16bit
    • 由于英文符号只需要用到低8位, 所以其高8位永远是0, 造成了空间上的浪费
  3. Character set VS. Character encoding
    “In there two terms, ‘set’ refers to the set of characters and their numbers (code points), and ‘encoding’ refers to the representation of these code points”
    Encodings: UTF-8, UTF-16
    Pooling for String objects:
    There is no need for maintaining serveral copies of String objects with the same literal, since String objects are immutable.
    But, when String objects created with “new” will not use the intern pool.

compareTo(String s2):

  1. 0 if the s1 is equal to s2.
  2. < 0 if the s1 is lexicographically less than s2

  3. 0 if the s1 is lexicographically greater than s2 (more characters)
    Definition of lexicographic ordering:

     if s1.length() != s2.length(): s1.length() - s2.length()
 else:                            s1.charAt(k) - s2.charAt(k)

String problems in Java == 本质为In-place modify a char[].
String: Immutable. 每次加加减减都会导致重新new一个String, 开销巨大, 只要对String有改变操作就不用
使用任何一个String API之前, 必须了解它的时间空间复杂度:
尽量避免调用deleteCharAt(i), subString(start, end), String1 + String2, because they may use O(n) time and/or O(n) extra space.

SubString -> strstr: determine whether a String is a subString of another String

  1. Regular method (Brute Force)
    Try all the possible substring in source that has length equals to pattern.length and then check whether their contents match.
    Time: O(m * n)
  2. Rabin-Carp (hash based string matching):
    Principle: If we can hash the short string to a unique Integer (via a hash function without collision), then we can just compare each subString of s1’s hashed value and compare it with s2’s hash value.
    1. Remove the leftmost item from the polynomial function
    2. All the rest items of (bc’s hashed value) * 26
    3. add new item d

Reversal(swap)

  1. apple -> elppa

    For given index k, rotate part1:[0, length-k) with part2:[length-k, length-1]

    2. Reverse(array, 0, k - 1);
    3. Reverse(array, k, n - 1);

Encoding/decoding: aaaabcc -> a4b1c2 (Run length Encoding)

Restriction: in-place (if not in-place, we can use a StringBuilder to easily perform)
Similar to Char Replacement(be careful about two cases: become longer / shorter)
Data Structure:
- s: the letters to the left of s (not including s) are the processed letters that should be kept
- f: the character being processed
- Initialization: s = f = 0;
Step1:
Deal with the cases where the adjacent occurrence of the letters >= 2, which will make the original String shorter;
In this case, the length of array would be the same or shorten.
Step2:
Deal with the cases where the adjacent occurrence of the letter == 1;
In this case, the length of array would be lengthen.
Step3:
Finalize the String by calling resize();

Sliding windows using slow/fast pointers

Sliding windows:
slow: the left border of the window;
fast: the right border of the window;
HashMap: <char key = letter, int value = the frequency of the letter>

  1. Longest subString that contains only unique chars ‘abcda’ return 4;
    Data Structure:
    slow: the left border of the window;
    fast: the right border of the window;
    HashMap: <char key = letter, int value = the frequency of the letter>
  2. Find all anagrams of a subString S2 in a long String s1
    Data Structure:
    s: the left border;
    f: the right border;
    HashMap: the number of occurrences in array[a…f] of each character(ignoring characters that are not in S2)
    numMatchingChars: number of matching characters
  3. Given a 0-1 array, you can flip at most k ‘0’s to ‘1’s. Please find the longest subArray that consists of all ‘1’s.
    Actually, it is the longest subArray that contains at most k zeros.(Same as 4.1)
