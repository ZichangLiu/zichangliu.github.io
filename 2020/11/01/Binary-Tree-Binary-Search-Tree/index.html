Binary Tree & Binary Search Tree

Posted on 2020-11-01 In Data Structure Disqus:

1. Basic Concepts

Key Point: 和LinkedList一样, 不要失去对root的控制权

Tree traversal

1. pre-order
2. in-order
3. post-order

Recursive: 区别就是何时打印自己的value
Trick: Base case usually refers to the null ChildNode below the leaf node.
Iterative: 防止树高过大导致call stack overflow
The unfinished methods must be retained in STACK before it can return.
Although it is not easy to convert all recursive solution to a ‘while loop’ soluton without any auxilliary data strcuture’s help.
1. preorder: pollFirst栈顶, 右进去, 左进去
因为FILO, 右孩子会一直被压在栈底直到左子树全部输出完毕, 直到栈变空, 意味着root的左右子树全部打印完毕 (相当于把stack的消耗转移到了heap上)
2. inorder: 问题变成了我们不能将根直接pollFirst, 因为需要先遍历左子树
额外需要一个helper来帮助判断左子树是否已经遍历完成, if helper==null, let helper == stack.pollFirst(); then helper = helper.right
3. postorder:
trick: reversed postorder traverse is just the modified version of preorder, for right-root-left, but space O(n) not O(h);

Three Kinds of Binary Tree

Balanced binary tree:

任意节点的左右子树高度差为0或1
结论: 树高一定为O(logn)
–Complete binary tree:
除了最后一层的右侧可以有空节点, 必须全满

Binary Search Tree:

For every node in the BST, the values in its left subtree are all smaller than its value, and the values in its right subtree are all larger than its value.
-Sorted tree, if print in in-order sequence, we can have an ascending order array.
-What if dulicate values?
class TreeNode {int value; int counter; left & right}

Self-balancing Binary Search Tree:

search(), insert(), remove(), operations are all guaranteed to be O(logn), rather than O(height)
Eg. AVL Tree, Red-black Tree (TreeMap/TreeSet)
Just a tool, no need to know about implmentation.

More Discussion:

Binary tree是最常见和recursion结合最紧密的面试题型
原因:

  1. 每层的node具备的性质, 传递的值和下一层往往一致, 易于定义recursive rule
  2. Base case (generally): null pointer under the leaf node

2. 相关题型:

Recursion在tree题目的基本应用大致分为两种:

  1. 把value从上往下传递, then从下往上
    1. BST判定方法
  2. 只把value从下往上传递(常见)
    1. getHeight(Node root): 自底向上传integer value
    2. isBalanced(Node root): 自底向上传boolean value
    3. isSymmetric(Node root): 自底向上传boolean value
    4. Assign the value of each node to be the total number of nodes that belong to its left tree: 自底向上传integer value

Tail Recursion:
The recursive call is always the last execution statement. We can easily transfer the tail recursive to itertive solution as a optimiation to avoid call stack overflow.

BST的三种基本操作

  1. Search in BST
  2. Insert in BST
    新插入的节点一定是新树的one of leafs
    When the tree structure could be changed, return a new root is a common way(like recursion way of reversing a linkedList)
    1. 怎么往下走
    2. 怎么利用下层返回回来的东西(挂上)
    3. 怎么把操作完的结果无差别的返回回去
  3. Delete in BST
    待删除节点可能为原树中任意节点:
    1. 叶子结点, 或只有一个孩子 (直接将自己的唯一的孩子接在自己父节点下)
    2. 有两个孩子的parent节点
      a. 将右子树中最小节点返回到被删除节点, 将其余右子树全部接在新节点下
      b. 如果右子树root没有左孩子, 即最小节点就是root, 则特殊处理一下

基本题型:

  1. Get the height of a binary tree.
    递归树和输入树相同, time: O(n), space: O(n)(糖葫芦)
  2. Check If Binary Tree Is Balanced
    time: O(nlogn) (每层time为n, 共有logn层)
  3. Symmetric Binary Tree
    递归树和输入树不同, time: O(n/2) = O(n), space: O(height)
