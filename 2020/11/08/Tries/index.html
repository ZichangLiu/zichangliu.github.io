Introduction

Definition

Tries - “Retrieve”

Example of Search Tree we are already very familiar with: Binary Search Tree

  • The keys are associated with each node in the tree
  • The keys can be any comparable type
  • Maintains the order of the keys by structural property (leftsubtree < root < rightsubtree)

Implementation:

A first glance at Trie:

一个字母一条边, 每个节点标注boolean isWord
Tries are different from binary search trees:

  • A key is associated with a “path from the root” to a node, instead of associated with a node
  • the edges from a node represent distinct characters, meaning “the possible next chars”
  • the key is string or sequence typed

The path from the root to each node is a prefix of a word in the dictionary
因此也叫前缀树

Complexity Analysis

Time Complexity: guarantee O(m)
Space Complexity:
If the trie is dense, since reusing the common prefix as many as possible, less space required. worst case O(nm), but usually much better than this.

Trie’s Drawbacks:

Time: If stored on disk, more random disk accesses (very expensive)
Space:

  1. Every TrieNode has extra space consumption -> extra space usage
  2. Memory allocation fragmentation
    especially when the Trie is sparse (the reusage is bad).

Use Case:

  1. string or sequence typed elements
  2. fast worst-case search/add/delete
  3. grouping common prefix, both for time/space efficiency
  4. problems related to prefix/matching

Basic Operations:

Search “cat” in the Trie.

  • finding the path from root which is equal to “cat”
  • for each of the character in “cat”, see if there is an edge associated with it for the next level.

Insert “carp” into the Trie.

  1. finding the path from root which is equal to “carp”
  2. for each of the character in “carp”, see if there is an edge associated with it for the next level.
    • case1: if there is an edge, go to the corresponding child node.
    • case2: else, create the child node

Delete “apple” from the Trie

  1. Finding the path from root which is equal to “apple”
  2. We can actually delete the two blue dotted line enclosed nodes

Solution 1:

  1. Store the path from root to the to-delete node in a stack.
  2. When popping a node from the stack, erase the node if it’s not a word and has no children.
  3. Could use recursion too.

DFS + Trie

Prefix Related Operations:

  • find all the words with a given prefix.

findAllWithPrefix(“ca”) = [“cap”, “cat”, “cathy”].

step 1 - find the node x associated with prefix “ca”
step 2 - do DFS on the subtree rooted at x to find all

It can be extended to support more operations, what other possible queries can it support?
Wildcard expression matching queries
regular expression matching queries
words within certain edit distance

e.g. Wildcard:

  • ‘?’ - means single character // .
  • ‘*’ - means >= 0 arbitrary characters. // >=0
    Question: find all words matching the expression with wildcard ‘?’ in the dictionary
    ‘?’ - can match any single character.

Example:

  • allMatch(“ca?”) = [“cap”, “cat”]
  • allMatch(“??p”) = [“app”, “cap”]

e.g. match “??p”

  • 3 levels, each level is trying to match one of the character in the target.
  • what are the possible next level states?
    1. ‘?’ – all the children
    2. ‘p’ – the corresponding children of ‘p’

DFS + Trie (Hard Question)

Word Search (Boggle Game)

Given a matrix of characters, you can move from one cell to neighbor cell(up, down, left, right), on the same path, each cell can be used only once. Find all possible words that can be formed by a sequence of adjacent characters.
Input: String[]: dictionary = {"at", "app", "apple", "cap", "cat", "cathy", "dog"}
char[][] board:
A P P D
P P L G
O G E I
D K L M
Output: List<String>: {"app", "apple", "dog"}
Analysis:
Maybe we can work on a simpler problem first:
- What if only given one target word, and we are to find if the word is in the matrix.
apple “apple”

DFS基本思想：
1. what does the recursion tree store on each level? (the semantic for each level)
- We need to know the base case when we can stop from exploring deeper.
2. How many different states should we try to put on this level? (Based on the semantic, what are all the options we can go for the next level)

  1. how many levels? – length of pattern
  2. what do we do at each level?
    in level i (0 <= i < length), see if (x,y) matches pattern[i]
  3. what are all possible ways for each level going the next
    if match, then explore four directions
    otherwise, return

pattern = APPPPLE
(0,0)
/ | | \ level 0: match pattern[0]
(-1,0) (1,0) (0,-1) (0,1)
/ | | \ \\ level 1: match pattern[1]
(0,0) (2,0) (1,-1) (1,1)
/ | | \ level 2: match pattern[2]
(0,1) (2,1) (1,0) (1,2)
level 3

Remember we need can not have duplicate cells on the same path(no cycle):

  • we need to record the cells traversed on the current DFS path.

Solution1: (brute force: dfs + backtracking)

Time complexity: m * n matrix, length of the word is L, size of the dictionary is k.
O(kmn* 4 ^ l) We are traversing all the paths instead of all the nodes.

Systematic way of solving problem:
From the existing solution, trying to find out the inefficient part and improve it by thoroughly understanding the solution and analysis on it.

Basically two types inefficiency in terms of Time:

  1. Unnecessary computation
    • binary search/BST
    • pruning in DFS/recursion
  2. Repeated computation
    • DP

Solution2: DFS + Trie

What is the inefficient part of our current solution? How can you improve it?

  • [app, apple]

group all the common prefix together, and only searched for once –> we need to use Trie to represent the whole dictionary. –> DFS on Trie.
Step 0: build the trie

  • Case 1: board[x][y] is in cur.children, explore four directions
  • case 2: … is not in cur.children, return Time = O(k * l + m * n * 4^l * l)
  1. how many levels? – height of trie.
  2. what do we do at each level? try to match current cell with current trie node’s children
  3. what are all possible ways for each level going the next:

Time complexity:
- m * n matrix
- length of the word is l
- size of the dictionary is k
Naive Solution: O(k * m * n * 4 ^ l)
Optimized Solution: O(k * l + m * n * 4 ^ l)
