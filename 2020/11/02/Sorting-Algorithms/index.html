Three sorting algorithms will be discussed:

  1. Seletion Sort
  2. Merge Sort

2. Seletion sort

2.1 Time And Space Complexity

Time = O(n^2) Space = O(1)

2.2 Algorithm

每次从未排序部分的数中找出最小的数并放在已排序部分的末尾
outer loop: how many iterations
inner loop: find the global min from the rest elements
swap the a[globalMin] with a[i]

3. Merge sort

3.1 Time And Space Complexity

Time = O(n) + O(nlogn) = O(nlogn) Space = O(n)

3.2 Algorithm

自顶向下: 不断分成两半知道分到每个小数组里只含一个元素(logn层)
Time = O(n) Space = O(n)
自底向上: 两两合并(logn层)
Time = O(nlogn) Space = O(n)

4. Quick Sort

4.1 Time And Space Complexity

Average = O(nlogn)
Worse case = O(n^2)

4.2 Algorithm

Key point: 挡板思想

  1. Pivot is randomly selected.
  2. Move povit element to the rightmost position.
  3. Partition: Iterate over the whole array, and put all numbers smaller than pivot to the left, then put pivot following (all numbers larger than or equal to pivot are already on pivot’s right-hand)
  4. Recursion.
    Recursive rule: QuickSort all numbers to the left of pivot and quickSort all numbers to the right of right of pivot.

两个挡板i,j三个区域a)b)c)的思想:
有两种不同元素

  1. [0…i]: i的左侧(不包含i)全部为比pivot小的数
  2. [i…j]: i和j之间为未知探索区域
  3. (j…n-1]: j的右侧(不包含j)全部为比pivot大或等于的数

4.3 Rainbow Sort:

三个挡板i,j,k四个区域同向/反向而行:
有三种不同元素
i = 0 -> :[0, i): i的左侧(不包含i)全部为”a”s
j = 0 -> :[i, j): j的左侧(不包含j)全部为”b”s
[j,k]: j和k之间为未知探索区域
k = n-1 ->:(k, n-1]: k的右侧(不包含k)全部为比”c”s
